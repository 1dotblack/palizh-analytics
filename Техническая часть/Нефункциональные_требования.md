# Нефункциональные требования (черновик)

## 1. Производительность и масштабируемость

- Ориентир по времени ответа для основных операций (каталог, корзина, оформление заказа) — уточнить.
- Подход к масштабированию:
  - горизонтальное масштабирование контейнеров backend/frontend,
  - кэширование (Redis) и оптимизация запросов к 1С.

## 2. Доступность и отказоустойчивость

- Требуемые RPO/RTO — уточнить.
- Репликация БД, резервные копии, избыточность по зонам (при необходимости).

## 3. Безопасность

- Соответствие требованиям по защите персональных данных (152‑ФЗ и внутренние стандарты заказчика).
- Хранение секретов и ключей в защищённых сервисах.
- Логирование действий пользователей (аудит) — уровни детализации уточнить.

## 4. Поддерживаемость

- Наличие документации по API и архитектуре.
- Стандарты код‑ревью и quality‑gate в CI.
