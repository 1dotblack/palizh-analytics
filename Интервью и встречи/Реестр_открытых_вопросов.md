## Осталось закрыть до MVP

- **№3 (ЧТЗ 01):** заказ через файл — согласовать **спецификацию** (формат `Excel`/CSV, колонки, правила валидации по каталогу). **Реализация в интерфейсе** — **post-MVP** (см. ЧТЗ 01); ответы по №3 можно зафиксировать до релиза MVP, чтобы не переделывать контракт позже.
- **№18 (ЧТЗ 09):** детали интеграционного контракта 1С (JSON по объектам, ограничения API) — уточнять **после** базовой реализации HTTP-сервисов на 1С и проверки вызовов; до этого — опираться на ЧТЗ 09 и `Техническая часть/Интеграция_1С.md`.
- **№30 (ЧТЗ 01/06/07):** финальный формат показа цен в ЛК (каталог/карточка/корзина, правила отображения).
- **№34 (ЧТЗ 11):** состав данных из 1С для поиска и параметры настройки поискового сервиса.
- **№36 (ЧТЗ 09):** финальный перечень доп.документов (`TDS/MSDS`, паспорта безопасности, `СГР`) для контуров 1С/ЛК.
- **№37 (ЧТЗ 09/05):** техническая реализация внешних ID/GUID и сценарии идемпотентности/дублей.
- **№40 (ЧТЗ 08/09):** контракт поля 1С для доступности номенклатуры (повтор заказа/недоступные позиции).
- **№25 (ЧТЗ 02/05/09):** по [реестру документов для проектирования](Реестр_документов_для_проектирования.md) **осталось получить файлы:** образец **ТТН** (№3), **протокол разногласий** (№7). Остальной набор в `Примеры документов/` получен; контракт обмена с 1С — не часть реестра образцов.
- **№47 (ЧТЗ 09):** подтверждение тестового контура 1С и процесса выдачи доступов.
- **№48 (ЧТЗ 09):** согласование схемы авторизации и сетевого доступа интеграции.
- **№49 (ЧТЗ 09):** формализация контракта файлов (MIME/размер/inline) — после рабочего API выдачи файлов на 1С (см. №18).
- **№50 (ЧТЗ 13):** фиксация НФТ-блока (RPO/RTO, нагрузка, 152-ФЗ, DDoS/WAF, окна обслуживания).

- **№12:** возврат в ЛК подтверждённых результатов претензии и связанных документов.
- **№24:** этапы и объём перевода клиентов на ЭДО.
