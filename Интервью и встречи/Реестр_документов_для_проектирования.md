# Реестр документов для проектирования

**Назначение:** перечень **печатных/файловых образцов** (PDF, DOCX и т.д.), которые нужны для проектирования **отображения и выдачи документов в ЛК** и понимания состава полей. Этот реестр **не** описывает контракт обмена с `1С`: форматы полей и вызовы API уточняются в ходе интеграции; рабочее решение — **данные приходят в JSON по API**. После получения образцов — обновлять статус и при необходимости переносить выводы в ЧТЗ 02, 05 и др.

**Связь:** [Реестр открытых вопросов](Реестр_открытых_вопросов.md) (вопрос № 25) — запрос образцов у клиента; [ЧТЗ 02 Документооборот](../ЧТЗ/02_документооборот.md), [ЧТЗ 05 Регистрация и онбординг](../ЧТЗ/05_регистрация_онбординг.md), [ЧТЗ 09 Интеграция с 1С](../ЧТЗ/09_интеграция_1С.md).

**По примерам заказов:** в OpenAPI зафиксированы поля `unit` (ед. измерения), `vatAmount` (НДС), `itemsCount` (кол-во позиций). Отдельный файл разбора снят; при необходимости повторить выводы — по PDF в папке и спецификации API.

**Чек-лист при изменении образцов заказов (`Заказ_*.pdf` или новых печатных форм):** сверить состав полей с `Техническая часть/openapi_mvp.yaml` и при расхождении обновить спецификацию и при необходимости ЧТЗ 01/08/09; зафиксировать дату изменения в коммите/заметке к реестру.

## Как использовать

1. **Запрос к клиенту:** использовать при интервью по документообороту (тема 4), онбордингу (тема 8) или отдельным запросом. В [Реестре открытых вопросов](Реестр_открытых_вопросов.md) заведён вопрос № 25 о сборе образцов.
2. **После получения:** обновить столбец «Статус» (Получен); описание формата и ограничений перенести в соответствующий ЧТЗ (02, 05, 09).
3. **Хранение образцов:** по договорённости с заказчиком — в папке проекта (например, `Интервью и встречи/Образцы документов/`) или ссылка на облако; в реестре указывать путь или «передан в ЧТЗ».
