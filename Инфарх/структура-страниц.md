# Структура страниц сайта (визуализация)

Визуализация информационной архитектуры Palizh B2B: дерево страниц и зоны доступа. Источник данных — [информационная-архитектура.md](информационная-архитектура.md) и [состав-и-задачи-страниц.md](состав-и-задачи-страниц.md).

---

## 1. Дерево страниц (sitemap)

Иерархия от точки входа: главная → разделы → подстраницы. Оформление заказа и ЛК доступны только авторизованному B2B-клиенту.

```mermaid
flowchart TB
    root["🏠 Главная"]

    subgraph Витрина["Витрина (гость + клиент)"]
        root --> K1["Каталог (розница)"]
        root --> K2["Каталог (опт)"]
        root --> K3["Каталог (бренды)"]
        K1 --> Карточка["Карточка товара"]
        K2 --> Карточка
        K3 --> Карточка
        root --> Обучение_публичное["Обучение"]
        Обучение_публичное --> О_курсе_публично["О курсе"]
        root --> О_компании["О компании"]
        root --> Новости["Новости"]
        Новости --> Страница_новости["Страница новости"]
        root --> Акции["Акции"]
        Акции --> Страница_акции["Страница акции"]
        Страница_акции --> Карточка
        root --> Поиск["Поиск"]
        root --> Избранное["Избранное"]
        root --> Обратная_связь["Обратная связь / Обращения"]
    end

    subgraph Заказ["Оформление заказа (только авторизованный)"]
        Карточка --> Корзина["Корзина"]
        Корзина --> Оформление["Оформление заказа"]
        Оформление --> Оплата["Оплата"]
        Оплата --> Уведомление["Уведомление об успехе / ошибке"]
    end

    subgraph Вход["Вход и регистрация"]
        root --> Авторизация["Авторизация / Регистрация"]
        Авторизация --> Политика["Политика конфиденциальности"]
        Авторизация --> Восстановление["Восстановление пароля"]
    end

    subgraph Служебные["Служебные элементы"]
        Куки["Куки-согласие (cookie consent)"]
    end

    subgraph ЛК["Личный кабинет (только авторизованный)"]
        ЛК_корень["Личный кабинет"]
        ЛК_корень --> Уведомления_ЛК["Уведомления"]
        ЛК_корень --> Обучение["Обучение"]
        Обучение --> О_курсе["О курсе"]
        ЛК_корень --> Заявки_на_обучение["Заявки на обучение"]
        ЛК_корень --> Документы["Документы"]
        ЛК_корень --> Обращения_ЛК["Обращения и претензии"]
        ЛК_корень --> Нестандартная_заявка["Нестандартная заявка / Связь с менеджером"]
        ЛК_корень --> Отложенная_корзина["Отложенная корзина"]
        ЛК_корень --> Профиль_компании["Профиль компании"]
        Профиль_компании --> Адреса_доставки["Адреса доставки"]
        ЛК_корень --> Избранные_товары["Избранные товары"]
        ЛК_корень --> История_заказов["История заказов"]
        ЛК_корень --> Оптовый_заказ["Оптовый заказ ?"]
    end

    Уведомление --> root
    Уведомление --> ЛК_корень
    Авторизация --> Корзина
    Авторизация --> ЛК_корень
```

*«?» — страницы/функции с открытыми вопросами.*

## 2. Зоны доступа

Кто что видит: гость (публично) vs авторизованный B2B-клиент.

```mermaid
flowchart LR
    subgraph Публично["Публично (гость)"]
        direction TB
        G1[Главная]
        G2[Каталоги: розница, опт, бренды]
        G3[Карточка товара]
        G8[Обучение]
        G4[О компании, Новости, Акции]
        G5[Поиск, Избранное]
        G6[Обратная связь]
        G7[Авторизация / Регистрация]
    end

    subgraph Только_клиент["Только авторизованный B2B-клиент"]
        direction TB
        C1[Корзина]
        C2[Оформление заказа]
        C3[Оплата]
        C4[Личный кабинет]
        C5[История заказов, Документы]
        C6[Профиль компании, Адреса доставки]
        C7[Обращения и претензии]
        C8[Отложенная корзина, Обучение, Заявки на обучение]
    end

    Публично --> |"Вход"| Только_клиент
```

## 3. Поток заказа (линейно)

От каталога до уведомления — только для вошедшего пользователя.

```mermaid
flowchart LR
    A[Каталог] --> B[Карточка товара]
    B --> C[Корзина]
    C --> D[Оформление заказа]
    D --> E[Оплата]
    E --> F[Уведомление]
    F --> G[Главная / ЛК]
```
