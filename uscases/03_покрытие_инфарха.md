# Покрытие сценариев страницами Инфарха

## Назначение

Сопоставление ID сценариев из `uscases/` с **узлами дерева страниц** в информационной архитектуре. Имена страниц и иерархия — источник истины: [Инфарх/структура-страниц.md](../Инфарх/структура-страниц.md), детали разделов — [Инфарх/состав-и-задачи-страниц.md](../Инфарх/состав-и-задачи-страниц.md).

При расхождении с макетом: сначала сверить с Инфархом, затем обновить этот файл или артефакты в `Инфарх/`.

## Авторизация и регистрация (AUTH)

| Сценарий | Страница / раздел Инфарх | Комментарий |
| --- | --- | --- |
| AUTH-01 | Авторизация / Регистрация | Форма «Стать клиентом», CTA с витрины |
| AUTH-01 | Главная | Точка входа к CTA (при размещении на главной) |
| AUTH-02 | — | Обработка в `1С` / админке / почта; не дублировать как страница клиента без отдельного решения |
| AUTH-03 | Авторизация / Регистрация | Вход |
| AUTH-03 | Личный кабинет | После успешного входа |
| AUTH-04 | Авторизация / Регистрация | Ссылка «Забыли пароль» |
| AUTH-04 | Восстановление пароля | Отдельный экран по сценарию |
| AUTH-05 | Авторизация / Регистрация | Конфликт дубликата (модал или подстраница по UX) |
