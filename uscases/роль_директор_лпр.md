# Роль: директор / ЛПР

## Назначение роли

Контроль заказов, документов и ключевых показателей взаимодействия с поставщиком.

## Границы доступа

- `MVP`: просмотр заказов, документов, обращений, раздела обучения.
- `MVP`: без админских системных настроек.
- `Post-MVP`: расширенная аналитика и согласовательные процессы.

## Use cases (MVP)

| ID | Предусловия | Основной поток | Альтернативы/ошибки | Результат | Источник данных | Уведомления | Критерии приемки (QA) |
|---|---|---|---|---|---|---|---|
| DIR-01 | Директор авторизован | Открывает историю заказов -> фильтрует по статусу/дате -> открывает заказ | Нет заказов -> пустое состояние | Видит актуальный список заказов | `1С` | Email/push по правилам релиза | Статусы заказа совпадают с `1С` |
| DIR-02 | Есть заказ с документами | Открывает документы -> скачивает нужный файл | Документ еще не готов -> корректный статус | Получен документ | `1С` | Email о готовности (если настроено) | Тип и реквизиты документа валидны |
| DIR-03 | Есть вопрос по заказу | Инициирует обращение/чат | Канал недоступен -> fallback на email | Обращение зарегистрировано | Платформа + ручной контур | Email внутренней маршрутизации | Обращение получает ID и статус |
| DIR-04 | Доступен раздел обучения | Просматривает программы -> принимает решение о записи сотрудников | Нет подходящих программ -> пустое состояние | Решение по обучению принято | Платформа | Напоминания по настройке | Список программ соответствует опубликованному контенту |
