# Матрица ролей и зон ответственности

## Роли в контуре

- Гость витрины (неавторизованный пользователь)
- Директор / ЛПР клиента
- Закупщик
- Бухгалтер
- Мастер-аккаунт клиента
- Менеджер по сопровождению (внутренняя роль)
- Контент-менеджер
- Поддержка / оператор
- Оператор админки
- Суперадмин

## MVP: границы доступов

- **Гость витрины**: просмотр витрины, каталога, карточек, новостей, акций, обучения; без корзины B2B-оформления.
- **Директор / ЛПР**: обзор заказов и документов компании, контроль статусов, согласование действий.
- **Закупщик**: работа с каталогом, корзиной, заказами, повтором заказа, связью с менеджером.
- **Бухгалтер**: документы, акты сверки, статусы оплаты, коммуникация по документам.
- **Мастер-аккаунт клиента**: управление пользователями компании и ролями (базовый контур).
- **Менеджер по сопровождению**: внешний коммуникационный контур (чат/почта/ручные процессы вне ЛК).
- **Контент-менеджер**: новости, акции, баннеры, контентные блоки.
- **Поддержка / оператор**: маршрутизация обращений/уведомлений, мониторинг входящих заявок.
- **Оператор админки**: драфт роли, уточнение матрицы post-MVP.
- **Суперадмин**: драфт роли, системные настройки и права post-MVP.

## Post-MVP (требует детализации)

- Роли "только просмотр", "логист клиента", "технолог клиента".
- Расширенная матрица прав админки по операциям.
- Дополнительные каналы внутренних уведомлений (кроме email).

## Источники истины

- Покрытие сценариев (`SCN-*`, `AUTH-*`, потоки дизайна) страницами Инфарха: [`03_покрытие_инфарха.md`](03_покрытие_инфарха.md).
- Коммерческие условия, документы, статусы заказа: `1С`.
- Витринный и маркетинговый контент: `платформа` (админка).
- Часть сопровождения (претензии/согласования): `ручной процесс` + внешние системы.
