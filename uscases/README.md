# USCases: роли и сценарии

Папка содержит role-based сценарии для трех целей:

- дизайн интерфейсов и пользовательских потоков;
- разработка функционала и интеграций;
- тестирование (приемка и регресс).

## Состав

- `00_матрица_ролей_и_зон_ответственности.md` — границы ролей и доступов (MVP / post-MVP).
- `01_сквозные_mvp_сценарии.md` — сквозные цепочки от витрины до исполнения.
- `03_покрытие_инфарха.md` — матрица «сценарий / поток → страница Инфарх».
- `сценарии_авторизация_и_регистрация.md` — единый контур онбординга, активации и входа.
- `роль_гость_витрина.md`
- `роль_директор_лпр.md`
- `роль_закупщик.md`
- `роль_бухгалтер.md`
- `роль_менеджер_сопровождения.md`
- `роль_контент_менеджер.md`
- `роль_поддержка_оператор.md`
- `роль_мастер_аккаунт_клиента.md` (драфт)
- `роль_оператор_админки.md` (драфт)

### Потоки для дизайна

Папка [`потоки_для_дизайна/`](потоки_для_дизайна/README.md) — сквозные потоки по **экранам и состояниям** (успех, ошибка, пусто, загрузка) для макетов; ссылки на `AUTH-*`, `BUY-*`, `SCN-*` и ЧТЗ/Инфарх.

## Правила использования

- Для каждой роли явно разделять `MVP` и `post-MVP`.
- Источник данных указывать явно: `1С`, `платформа`, `ручной процесс`.
- В QA-части использовать критерии приемки в формате "Дано/Когда/Тогда".
- При изменении ЧТЗ синхронизировать соответствующий файл роли и сквозные сценарии.
- Единый формат сценариев: `ID`, `Предусловия`, `Основной поток`, `Альтернативы/ошибки`, `Результат`, `Источник данных`, `Уведомления`, `Критерии приемки (QA)`.
- В каждом role-документе обязательно указывать блок `Связь с ЧТЗ`.
