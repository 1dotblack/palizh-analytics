# Роль: закупщик

## Назначение роли

Основной пользователь заказа: подбор товаров, корзина, оформление и повтор.

## Границы доступа

- `MVP`: каталог, карточки, корзина, оформление, история, повтор заказа.
- `MVP`: **внешний** чат-виджет (вопросы); **форма** нестандартной заявки / обращений по ТЗ.
- `Post-MVP`: расширенные бонусные и финансовые сценарии.

## Use cases (MVP)

| ID | Предусловия | Основной поток | Альтернативы/ошибки | Результат | Источник данных | Уведомления | Критерии приемки (QA) |
|---|---|---|---|---|---|---|---|
| BUY-01 | Закупщик авторизован | Ищет товар -> открывает карточку -> добавляет в корзину | Нет выдачи/товар недоступен | Корзина сформирована | `1С` (каталог/цены) | Email по настройке (ЧТЗ 10) | В корзине корректные позиции и цены |
| BUY-02 | Корзина сформирована | Заполняет оформление -> подтверждает заказ | Ошибка в данных/недоступна доставка | Заказ передан в обработку | `1С` + платформа | Подтверждение заказа | Заказ появляется в истории со статусом |
| BUY-03 | Есть прошлый заказ | Нажимает "Повторить заказ" -> система собирает корзину | Недоступные позиции помечены | Новая корзина для повтора | `1С` | Стандартные уведомления | Недоступные товары не ломают поток |
| BUY-04 | Функция включена | Загружает файл заказа -> проверка структуры -> подтверждает | Некорректный формат файла | Черновик/заказ создан | Платформа + `1С` | Ошибки валидации по email/в UI | Валидация колонок и позиций работает |
| BUY-05 | Нужен нестандартный товар | Открывает форму ЛК → отправляет описание/файлы | Неполные данные заявки | Заявка зафиксирована, email менеджерам | Платформа + ручной контур | Email по маршрутизации админки | Подтверждение в UI; при наличии списка — статус `Отправлено` |
