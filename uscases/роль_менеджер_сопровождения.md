# Роль: менеджер по сопровождению

## Назначение роли

Сопровождает клиента вне ЛК-контуров: согласования, консультации, ручные процессы.

## Границы доступа

- `MVP`: чат/обращения, согласование нестандартных заявок, сопровождение претензий.
- `MVP`: работа через платформу + внешние каналы (email/телефон).
- `Post-MVP`: углубленная автоматизация внутреннего workflow.

## Use cases (MVP)

| ID | Предусловия | Основной поток | Альтернативы/ошибки | Результат | Источник данных | Уведомления | Критерии приемки (QA) |
|---|---|---|---|---|---|---|---|
| MGR-01 | Есть входящее обращение | Менеджер открывает обращение -> отвечает клиенту -> фиксирует статус | Недостаточно данных | Клиент получает ответ | Платформа + ручной процесс | Email клиенту/внутри команды | Ответ отправлен и зафиксирован |
| MGR-02 | Получена нестандартная заявка | Проверяет данные -> маршрутизирует в нужный контур | Неполная заявка | Заявка передана ответственному | Платформа | Email ответственному | Маршрут и ответственный назначены |
| MGR-03 | Есть претензия со статусом `Отправлена` | Проверяет комплект -> ведет коммуникацию во внешнем контуре | Нет обязательных материалов | Клиенту дано требование на доукомплектование | Платформа + внешние системы | Email по этапам | Претензия не теряется и имеет владельца |
| MGR-04 | Есть вопрос по доставке/документам | Уточняет данные в `1С` -> отвечает клиенту | Данные в `1С` задерживаются | Клиент информирован о сроках | `1С` + ручной процесс | Email/чат | Ответ согласован с фактическими данными `1С` |
