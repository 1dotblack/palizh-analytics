# Потоки для дизайна

Документы в этой папке описывают **сквозные пользовательские потоки по экранам**: что показать, в каком порядке, какие состояния (успех, ошибка, пусто, загрузка).  
Дополняют role-based файлы в `uscases/` и привязаны к ЧТЗ и Инфарх.

## Как пользоваться

1. Открыть нужный поток — таблица **шаги для макета** = последовательность экранов/состояний.
2. Сверить с [Инфарх/состав-и-задачи-страниц.md](../../Инфарх/состав-и-задачи-страниц.md) и макетом в Pixso (карта сайта).
3. Связь с тестами: ID шагов и ссылки на `AUTH-*`, `BUY-*`, `SCN-*` в родительских документах `uscases/`.

## Шаблон одного потока

- Цель, роли, `MVP` / `post-MVP`
- Связанные USCase / сквозные сценарии
- ЧТЗ и Инфарх
- Шаги для макета (экран → действие → контент → источник данных)
- Обязательные UI-состояния: пусто, загрузка, ошибка, успех
- Открытые вопросы для визуала
