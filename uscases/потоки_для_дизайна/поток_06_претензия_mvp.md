# Поток для дизайна: претензия (MVP)

## Цель

Подать претензию по заказу с вложениями и получить подтверждение со статусом «Отправлена»; дальнейшая обработка вне платформы.

## Роли

- Закупщик / директор (кто имеет право подачи по ЧТЗ 07)

## Контур

- `MVP`: выбор заказа/отгрузки → форма по типу претензии → вложения → отправка → карточка со статусом `Отправлена`.
- `Post-MVP`: расширенные статусы и документы в ЛК.

## Шаги для макета

| № | Экран / состояние | Действие пользователя | Что показать | Источник данных |
|---|-------------------|------------------------|--------------|-----------------|
| 1 | Точка входа | Из заказа или раздела «Претензии» | CTA «Подать претензию» | Платформа |
| 2 | Выбор заказа/отгрузки | Выбирает из списка | Подстановка реквизитов и позиций | `1С` |
| 3 | Тип претензии | Выбирает тип (бой, недопоставка, излишек, качество) | Условные поля по типу (см. ЧТЗ 04) | Платформа |
| 4 | Вложения | Загружает фото/файлы | Лимиты форматов, прогресс | Платформа |
| 5 | Подсказки | — | Для боя + ТК: напоминание про коммерческий акт | Платформа |
| 6 | Отправка | Подтверждает | Экран успеха, ID, дата, статус `Отправлена` | Платформа |
| 7 | История | Просматривает список претензий | Карточки с минимальным статусом MVP | Платформа |

## UI-состояния

- Валидация обязательных полей по типу.
- Ошибка загрузки файла.
- Недоступность подачи (нет подходящей отгрузки) — пустое состояние с пояснением.

## Открытые вопросы для дизайна

- Финальный список обязательных полей после UX-проектирования формы (см. ЧТЗ 04).
- Формат Rub-Out и прочие вложения — уточнить подписи и подсказки.
