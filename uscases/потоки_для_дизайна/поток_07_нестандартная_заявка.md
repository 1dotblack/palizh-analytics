# Поток для дизайна: нестандартная заявка (колеровка / оборудование)

## Цель

Отправить заявку менеджеру с типом, описанием и файлами без оформления как обычного заказа из каталога.

## Роли

- Закупщик (основной)

## Контур

- `MVP`: форма заявки → подтверждение отправки → уведомление менеджеру (внутренний контур).
- `Post-MVP`: статусы заявки в ЛК (если появятся в ТЗ).

## Шаги для макета

| № | Экран / состояние | Действие пользователя | Что показать | Источник данных |
|---|-------------------|------------------------|--------------|-----------------|
| 1 | Точка входа | Из каталога / футера / «Связь с менеджером» | Кратко: для чего форма | Платформа |
| 2 | Тип заявки | Выбирает колеровку / оборудование | Разные подсказки под тип | Платформа |
| 3 | Описание | Вводит текст, при необходимости привязку к товару | Текстовое поле, опциональный выбор из каталога | Платформа + `1С` |
| 4 | Файлы | Прикрепляет PDF/Excel/Word | Список файлов, лимиты | Платформа |
| 5 | Отправка | Отправляет | Успех: «Заявка отправлена», что будет дальше (менеджер свяжется) | Платформа |
| 6 | Ошибка | — | Не отправилось — повтор / поддержка | Платформа |

## UI-состояния

- Неполная форма — подсветка обязательных полей.
- Превышение размера/типа файла.

## Открытые вопросы для дизайна

- Нужна ли отдельная страница «История нестандартных заявок» в MVP или только подтверждение отправки.
