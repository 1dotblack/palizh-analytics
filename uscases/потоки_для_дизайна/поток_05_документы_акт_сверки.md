# Поток для дизайна: документы по заказу и акт сверки

## Цель

Найти и скачать документы по заказу; инициировать запрос акта сверки.

## Роли

- Бухгалтер
- Директор (просмотр)

## Контур

- `MVP`: список/карточка документов в ЛК, скачивание PDF; запрос акта сверки в ручной контур.
- `Post-MVP`: ЭДО из ЛК.

## Связь с USCases

- `роль_бухгалтер.md`: `ACC-01`, `ACC-02`, `ACC-03`
- `роль_директор_лпр.md`: `DIR-02`
- `01_сквозные_mvp_сценарии.md`: `SCN-02`

## Шаги для макета

| № | Экран / состояние | Действие пользователя | Что показать | Источник данных |
|---|-------------------|------------------------|--------------|-----------------|
| 1 | Раздел «Документы» | Фильтрует по заказу / периоду | Список документов с типами (счёт, УПД, ТТН, ПК, акт) | `1С` |
| 2 | Деталь документа | Скачивает файл | Кнопка скачивания, метаданные (дата, номер) | `1С` |
| 3 | Документ не готов | — | Статус «формируется» / ожидание | `1С` |
| 4 | Запрос акта сверки | Заполняет период и отправляет | Форма запроса, подтверждение принятия | Платформа → ручной контур |
| 5 | Ошибка выгрузки | — | Сообщение + повтор / поддержка | Платформа |

## UI-состояния

- Пустой список документов для нового клиента.
- Долгая генерация — индикатор или асинхронное уведомление (по ЧТЗ 10).

## Открытые вопросы для дизайна

- Нужна ли группировка документов по заказам vs единая лента.
- Отображение доп. документов (TDS/СГР) — по ЧТЗ 02 и реестру документов.
